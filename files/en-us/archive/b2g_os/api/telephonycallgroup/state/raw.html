The state read-only property of the TelephonyCallGroup interface returns a DOMString indicating the state of the call group.

Syntax

var conferenceCallState = myTelephonyCallGroup.state;

Value

A DOMString.

Call state values

State String Description
empty There is currently no active conference call.
connected The active conference call is connected.
held The active conference call is on hold.

Examples

// Create your Telephony object
var tel = navigator.mozTelephony;

// Make two calls; assume they are both answered
var call1, call2;
tel.dial(number1).then(call => call1 = call);
tel.dial(number2).then(call => call2 = call);

// Add both calls to the conference call
// Two entries will appear in conference.calls (call1, call2)
// but nothing in tel.calls
var conference = tel.conferenceGroup;
conference.add(call1, call2);

conference.state;
