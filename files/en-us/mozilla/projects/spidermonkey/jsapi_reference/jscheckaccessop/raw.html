JSCheckAccessOp is the type for . (It is also the type of the callback set by .)

Syntax

typedef JSBool
(* JSCheckAccessOp)(JSContext *cx, JSObject *obj, jsval id,
                    JSAccessMode mode, jsval *vp);
Name Type Description
cx The JS context in which the property access attempt is occurring.
obj The object whose properties are being accessed.
id The name or index of the property being accessed.
mode The type of access being checked.
vp Out parameter. On success, the callback must set *vp to the of the property.

Description

Check whether obj[id] may be accessed per mode, returning JS_FALSE on error/exception, JS_TRUE on success with obj[id]'s in *vp. As for , id is either a string or an int .

JSCheckAccessOp implementations generally work by using JSDBGAPI functions such as and to obtain the principals of the code attempting the checked operation, then examining those principals and comparing them with the system's security policy. The nature of principals and the security policy are entirely up to the application.

If a class leaves the checkAccess field NULL, a runtime-wide object access callback is called instead; see .
