As of SpiderMonkey 1.8.5, JS_SealObject has been removed from the JSAPI, because ECMAScript 5 includes a "seal" concept (namely, that of -->) which is quite different from that of JS_SealObject.  Moreover, JS_SealObject's exact semantics are subtly incompatible with ECMAScript.  --> and --> are near-substitutes compatible with ECMAScript semantics, implementing semantics compatible with those of -->.

Prevent modification of object fields.

Syntax

JSBool
JS_SealObject(JSContext *cx, JSObject *obj, JSBool deep);
Name Type Description
cx --> A context.
obj --> The object to seal.
deep --> If true, seal all objects directly or indirectly reachable from obj's slots.

Description

JS_SealObject prevents all write access to the object, either to add a new property, delete an existing property, or set the value or attributes of an existing property.
