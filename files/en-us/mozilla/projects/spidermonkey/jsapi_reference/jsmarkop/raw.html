JSMarkOp is the type of the JSClass.mark callback in SpiderMonkey 1.7 and earlier. In SpiderMonkey 1.8, JSMarkOp will be deprecated. All new code using SpiderMonkey 1.8 or later should use a --> instead to ensure that the tracing APIs work properly.

Syntax

typedef uint32
(* JSMarkOp)(JSContext *cx, JSObject *obj, void *arg);
Name Type Description
cx --> The JS context in which the mark phase of garbage collection is occurring.
obj --> The object to be marked.
arg void * Opaque data which the mark hook must pass through to -->.

Description

Call --> for each GC thing directly reachable from obj. Use --> to check whether a value needs to be marked and --> to convert the jsval to a pointer.

JSClass hooks

JSClass offers the following hook:

  • The JavaScript engine calls the --> callback during the mark phase of garbage collection.
